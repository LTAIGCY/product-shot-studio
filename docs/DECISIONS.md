# 重要决策记录

本文件记录会长期影响产品和工程的决策。新决策按日期追加，不覆盖历史原因。

## 2026-06：Windows 桌面端采用 Electron

**决定**：使用 Electron + React + TypeScript，首发 Windows 10/11。

**原因**：需要本地文件系统、批量素材、导出、密钥安全存储和可安装桌面体验，同时复用 Web UI 技术栈。

**影响**：文件和凭据操作放在主进程，渲染进程通过安全 IPC 调用；发布前需要 Windows 打包和干净机器验收。

## 2026-06：首版模型切换为国内供应商

**决定**：图片与视频生成优先接入阿里百炼、火山方舟和腾讯混元。

**原因**：目标用户主要在中国，需要降低跨境网络、地域可用性和账户风控的不确定性。

**影响**：模型能力通过统一适配层暴露，模型 ID 放在共享配置；提示词和错误信息统一中文化。

## 2026-06：本地素材与云端账本分离

**决定**：用户素材、图库、历史和 API Key 保留本机；账号、钱包、交易、任务摘要和在线状态进入云端后端。

**原因**：减少云存储成本和隐私范围，同时让积分与充值具备中心账本。

**影响**：换电脑后本地素材不会自动同步；云端数据库不能替代本地素材备份。

## 2026-06：密码只存带盐哈希

**决定**：数据库不保存或展示明文密码，后台仅支持密码重置。

**原因**：明文或可逆密码会造成不可接受的账户安全风险。

**影响**：用户忘记密码时只能重置，不能找回原密码；日志、后台和备份同样不得出现明文。

## 2026-06：SQLite 作为云端 MVP 中心账本

**决定**：初期单实例服务继续使用 Prisma + SQLite。

**原因**：当前用户规模小、部署简单，能快速验证账号、钱包和任务流程。

**影响**：上线前必须配置一致性备份和恢复演练；需要多实例、高可用或更高并发时迁移托管 PostgreSQL/MySQL。

## 2026-06：云端后端独立于桌面软件运行

**决定**：后端部署为腾讯云 systemd 常驻服务，通过 `api.qingpaiai.com` 和 HTTPS 对外提供 API。

**原因**：关闭开发电脑或桌面软件不能影响账号和账本服务。

**影响**：桌面端默认连接云端；本地后端只用于显式的隔离开发模式。

## 2026-07：GitHub `main` 是唯一稳定来源

**决定**：所有功能使用短期分支和 PR，合并后才允许部署生产；长期事实写入仓库文档。

**原因**：多人和 Codex 并行开发时，需要可审查、可回滚且一致的代码来源。

**影响**：服务器不直接修改源码；并行任务使用独立 worktree；聊天记录不替代文档和 Git 历史。

## 2026-07：协作者使用独立身份和密钥

**决定**：每位协作者使用自己的 GitHub 身份、SSH 密钥和服务器用户。

**原因**：共享私钥无法单独撤权，也无法准确审计操作来源。

**影响**：只交换公钥；私钥永不通过聊天、Git、网盘或即时通信发送。
