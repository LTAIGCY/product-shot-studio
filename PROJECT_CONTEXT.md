# 轻拍项目上下文

最后更新：2026-07-01

## 项目目标

轻拍（Product Shot Studio）是一款面向 Windows 的 AI 商拍工作流软件。用户可以导入普通产品照片，调用国内图片和视频模型生成电商展示素材，并在桌面端完成预览、编辑、图库整理、历史管理和导出。

当前稳定版本为 `v0.3.5`，稳定代码以 GitHub 仓库 `LTAIGCY/product-shot-studio` 的 `main` 分支为准。

## 已确认运行状态

- 桌面端：Electron + React + TypeScript，开发环境默认连接云端账本。
- 云端 API：`https://api.qingpaiai.com`。
- 健康检查：`https://api.qingpaiai.com/health`。
- 监测后台：`https://api.qingpaiai.com/admin`。
- 展示官网：`qingpaiai.com`，当前用于“轻拍图片设计”作品展示和 ICP 备案。
- 后端服务：腾讯云 Ubuntu 上的 systemd 服务 `product-shot-studio`，由 Nginx 反向代理并启用 HTTPS。
- 云端数据库：服务器 `server/data/product-shot-studio.db`，当前为 SQLite 中心账本。
- 本地数据：原图、生成素材、历史记录、画布项目、个人图库和模型 API Key 仍保存在用户本机。

## 数据边界

云端账本保存账号、密码哈希、钱包、交易、生成任务摘要、在线状态和审计事件。桌面端本机保存用户素材、历史、图库、画布数据、会话摘要和供应商 API Key。

密码不以明文存储或展示。API Key 使用本机安全存储，不进入 Git、日志或云端数据库。当前模型调用仍由桌面端直接请求供应商，因此商业化前需要迁移到后端代理，才能形成可靠的计费约束。

跨模块任务先在总控对话拆解，再交给专题对话。完成一个功能后，把稳定结论写回本仓库文档；聊天记录不作为唯一事实来源。

## 当前优先事项

1. 完成 ICP 备案，并在备案通过后补齐官网备案号和正式 HTTPS 验收。
2. 设计真实充值订单和微信支付闭环，包括回调验签、幂等、退款与对账。
3. 为 SQLite 账本建立自动备份和恢复演练，再评估迁移托管数据库。
4. 将供应商模型调用迁到云端代理，避免桌面端绕过扣费。
5. 建立服务器自动部署、健康监测、日志轮转和告警。
6. 为协作者使用独立 SSH 密钥和 Linux 用户，不共享私钥。
